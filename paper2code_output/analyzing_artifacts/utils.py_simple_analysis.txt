 Here's the logic analysis for the `utils.py` file, which contains utility functions for logging metrics, handling configuration, and other miscellaneous tasks:

```python
import yaml
import logging
from typing import Any, Dict, Union

# Configuration handling
def load_config(config_path: str) -> Dict[str, Any]:
    """
    Loads the configuration from a YAML file.
    
    Args:
        config_path (str): The path to the configuration file.
    
    Returns:
        Dict[str, Any]: The loaded configuration.
    """
    with open(config_path, 'r') as file:
        config = yaml.safe_load(file)
    return config

# Logging setup
def setup_logging(log_file: str) -> None:
    """
    Sets up the logging configuration.
    
    Args:
        log_file (str): The path to the log file.
    """
    logging.basicConfig(
        filename=log_file,
        level=logging.INFO,
        format='%(asctime)s - %(levelname)s - %(message)s'
    )
    console = logging.StreamHandler()
    console.setLevel(logging.DEBUG)
    formatter = logging.Formatter('%(asctime)s - %(levelname)s - %(message)s')
    console.setFormatter(formatter)
    logging.getLogger('').addHandler(console)

# Metric logging
def log_metrics(metrics: Dict[str, float], log_file: str) -> None:
    """
    Logs the given metrics to the specified file.
    
    Args:
        metrics (Dict[str, float]): The metrics to log.
        log_file (str): The path to the log file.
    """
    with open(log_file, 'a') as file:
        for metric_name, metric_value in metrics.items():
            file.write(f"{metric_name}: {metric_value}\n")
    logging.info(f"Metrics logged: {metrics}")

# Risk level configuration
def get_risk_configuration(config: Dict[str, Any], risk_level: str) -> float:
    """
    Retrieves the risk level configuration from the given config.
    
    Args:
        config (Dict[str, Any]): The configuration dictionary.
        risk_level (str): The risk level identifier.
    
    Returns:
        float: The risk level threshold.
    """
    return config['evaluation']['ETCU_risk_levels'].get(risk_level, 0.0)

# Example usage
if __name__ == "__main__":
    # Load configuration
    config = load_config('config.yaml')
    
    # Setup logging
    setup_logging('application.log')
    
    # Example metrics
    metrics = {
        'smECE': 0.05,
        'ETCU': 0.8
    }
    
    # Log metrics
    log_metrics(metrics, 'metrics_log.txt')
    
    # Get risk configuration
    low_risk_threshold = get_risk_configuration(config, 'low')
    print(f"Low risk threshold: {low_risk_threshold}")
```

### Explanation of the Utility Functions

1. **load_config**: This function reads the configuration from a YAML file and returns it as a dictionary. This configuration is used to set up various parameters for data preprocessing, model training, and evaluation.

2. **setup_logging**: This function sets up the logging configuration to log messages with a specific format to both a file and the console. This is useful for debugging and tracking the execution of the program.

3. **log_metrics**: This function logs the given metrics to a specified file. Each metric is written as a separate line, and the function also logs the metrics to the console for immediate visibility.

4. **get_risk_configuration**: This function retrieves the risk level configuration from the given configuration dictionary. It is used to get the threshold for the Expected Tool-Calling Utility (ETCU) corresponding to a specific risk level.

These functions are designed to be flexible and reusable across different parts of the application, making it easier to manage configurations and logging.
